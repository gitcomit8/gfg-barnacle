The Bug Scenario

This module simulates a Server-Side Rendering (SSR) hydration mismatch bug where:

  1. The server renders HTML with random values (UUID, Math.random(), timestamp)
  2. The client-side JavaScript tries to "hydrate" the same component
  3. During hydration, the component generates NEW random values
  4. React detects a mismatch between server HTML and client virtual DOM
  5. Result: Event listeners don't attach properly - buttons won't work!

Root Cause Analysis

  • Math.random() returns different values on each execution
  • Date.now() returns different timestamps at different times
  • UUID generation creates unique IDs each time
  • All these are called during component render on BOTH server and client
  • React expects the initial render to produce identical output
