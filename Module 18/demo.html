This demo will show you how re-entrancy issues cause RefCell panics when JavaScript callbacks call back into Rust. The "Trigger Bug" button will crash the WASM module to demonstrate the issue.

1. Add Items SAFE

Add items to the processor. This operation is safe and won't trigger the bug.

    2. Safe Processing SAFE

    Process items with a callback that DOES NOT call back into Rust. This is safe.

    3. Buggy Processing BUG!

    Process items with a callback that tries to call get_item_count() from within the callback. This will trigger the re-entrancy bug and cause a panic!

    4. Query State SAFE (when not processing)

    Query the current state of the processor. Safe when not processing.
